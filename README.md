# Granularr

## Briefly

Granularr is an orchestrator. 
It tracks/receives a user's music taste using _last.fm_.
Allows users to create a playlist curated/managed/collaborated by Granularr.
Granularr uses songs available in Navidrome when it can. 
Granularr kindly asks Lidarr when it can't.

Granularr manages the playlist as the Navidrome user, so playlist is still owned by the same user and is compatible with all music players.
Granular can work alongside user-added songs in the playlist, only ever touching the ones it added, unless user wants otherwise.
When its time to for a refresh, Granularr asks Lidarr for new songs to come in, old boring ones to be discarded. 
Granular login is your Navidrome login, literally.

All *arr's in your setup keep doing what they do best.
No extra downloaders, indexers, metadata editors or other bs

 As simple as that.
